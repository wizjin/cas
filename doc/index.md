# Documentation Index

Use this index to load only the minimum documentation required for the task.

## Files

- `doc/build.md`: Build system, Make workflow, build directories, release output.
- `doc/cli.md`: CLI commands, dispatch rules, expected user-visible behavior.
- `doc/style.md`: C language rules, naming, formatting, comments, log constraints.
- `doc/testing.md`: Unit testing, coverage target, static analysis scope.
- `doc/dependencies.md`: Third-party dependency selection and storage rules.

## Reading Order

- Build or release tasks: read `doc/build.md`.
- CLI behavior changes: read `doc/cli.md`.
- Code edits or new modules: read `doc/style.md`.
- Test, coverage, or analysis tasks: read `doc/testing.md`.
- Dependency changes: read `doc/dependencies.md`.

Return to this file when the task changes scope.
